AWS Responsible AI - Key Concepts for AIF-C01

FAIRNESS
Fairness in AI means ensuring models do not produce biased or discriminatory outputs. AWS emphasizes evaluating models across demographic groups. Amazon SageMaker Clarify provides bias detection and explainability features. Bias can occur in training data, model design, or deployment context. Pre-training bias metrics include class imbalance and label imbalance. Post-training bias metrics include disparate impact and accuracy differences across groups.

TRANSPARENCY AND EXPLAINABILITY
Explainability refers to understanding why a model produces a given output. Amazon SageMaker Clarify provides SHAP (SHapley Additive exPlanations) values to identify feature importance. Model cards document model purpose, performance, limitations, and intended use. Transparency requires clear documentation of training data sources, model architecture, and evaluation methodology.

PRIVACY AND DATA GOVERNANCE
AI systems must protect personal data in training and inference. AWS recommends data minimisation - using only data necessary for the task. Amazon Macie can detect sensitive data in S3 buckets used for training. Differential privacy techniques add noise to prevent re-identification of individuals from model outputs. Data lineage tracking ensures accountability for datasets used in model training.

ROBUSTNESS AND SAFETY
Robust AI systems perform consistently across edge cases and adversarial inputs. Amazon Bedrock Guardrails provides content filtering, topic denial, word filters, and sensitive information redaction. Guardrails can be applied to both inputs and outputs of foundation models. Red-teaming involves adversarial testing to identify safety vulnerabilities. Models should degrade gracefully on out-of-distribution inputs.

HUMAN OVERSIGHT
Humans should remain in control of consequential AI decisions. AWS recommends human review workflows for high-stakes use cases. Amazon Augmented AI (A2I) enables human review of ML predictions. The principle of human-in-the-loop ensures critical decisions are reviewed by humans before action. Automation should be reserved for low-risk, reversible decisions.

ACCOUNTABILITY
Organisations are accountable for AI systems they deploy. AWS shared responsibility model applies to AI: AWS secures the infrastructure, customers are responsible for model behaviour and data. AI governance frameworks should define roles, policies, and escalation paths. Audit trails should capture model versions, input data, and outputs for review. Incident response plans should cover AI system failures.

MODEL EVALUATION
Evaluating generative AI requires new metrics beyond traditional accuracy. Human evaluation assesses relevance, coherence, and harmlessness. Automated evaluation uses reference-based metrics (BLEU, ROUGE) and model-based scoring. Amazon Bedrock Model Evaluation supports both automatic and human evaluation workflows. Evaluation should be ongoing, not just at deployment.

RESPONSIBLE AI DIMENSIONS (AWS FRAMEWORK)
AWS defines responsible AI across six dimensions:
1. Fairness - equitable treatment across groups
2. Explainability - understandable outputs
3. Privacy and Security - data protection
4. Safety - preventing harmful outputs
5. Controllability - human oversight and intervention
6. Veracity and Robustness - accurate and reliable outputs

BEDROCK SAFETY FEATURES
Amazon Bedrock Guardrails: configurable filters for harmful content categories (hate speech, violence, sexual content, misconduct). Topic denial: block specific topics from model responses. Grounding: reduce hallucinations by requiring responses to be grounded in provided context. Sensitive information filters: redact PII such as names, addresses, and credit card numbers from inputs and outputs.
